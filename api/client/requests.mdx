---
title: "Funding, Withdrawal, and Portfolio Change Requests"
description: "Client API request endpoints: funding deposits, fund withdrawals, portfolio change requests, and TR impersonation routes — all on port 3001."
---

Request endpoints cover all client-initiated operations that require backend approval or processing: depositing funds into an account, withdrawing cash or redeeming investments, and requesting a change to a portfolio's investment strategy. All endpoints require a valid JWT access token. Requests follow a lifecycle with statuses such as `PENDING`, `PENDING_APPROVAL`, `APPROVED`, `REJECTED`, and `PROCESSED`.

<Note>
Trading Representative (TR) users can initiate risk profile assessments and strategy changes on behalf of clients using the `/v1/tr/*` endpoints. These create requests that are pending the client's explicit confirmation before taking effect.
</Note>

---

## Funding requests

Funding requests represent a client's intent to deposit money into their robo account. Each request requires proof of transfer (a file upload).

### POST /v1/funding-requests

Submit a new funding deposit request. Accepts `multipart/form-data` because the proof-of-transfer file is uploaded in the same request.

**Authentication**: JWT required

```bash
curl -X POST http://localhost:3001/v1/funding-requests \
  -H "Authorization: Bearer <accessToken>" \
  -F "accountId=ACC-00000001" \
  -F "amount=10000" \
  -F "currency=SGD" \
  -F "proofOfTransfer=@/path/to/receipt.pdf"
```

#### Request body (multipart/form-data)

<ParamField body="accountId" type="string" required>
  The account ID to credit (e.g., `ACC-00000001`).
</ParamField>

<ParamField body="amount" type="number" required>
  Deposit amount in the specified currency.
</ParamField>

<ParamField body="currency" type="string" default="SGD">
  ISO 4217 currency code (e.g., `SGD`, `USD`).
</ParamField>

<ParamField body="proofOfTransfer" type="file" required>
  Bank transfer receipt. Accepted formats: JPEG, PNG, PDF, HEIC. Maximum size: 10 MB.
</ParamField>

#### Response

<ResponseField name="id" type="string" required>
  Funding request ID (e.g., `FND-00000001`).
</ResponseField>

<ResponseField name="status" type="string" required>
  Initial status of the request. Typically `PENDING`.
</ResponseField>

<ResponseField name="message" type="string" required>
  Confirmation message.
</ResponseField>

#### Error responses

| Status | Meaning |
|--------|---------|
| 400 | Missing required field or unsupported file type |
| 403 | Account does not belong to the authenticated user |
| 413 | File exceeds the 10 MB limit |

## Withdrawal requests

Withdrawal requests cover three distinct operations: withdrawing cash to a bank account, cancelling an existing pending request, and redeeming an investment holding.

### POST /v1/withdraw-requests/fund-withdrawal

Submit a cash withdrawal request to transfer funds from the robo account to a designated bank account.

**Authentication**: JWT required

```bash
curl -X POST http://localhost:3001/v1/withdraw-requests/fund-withdrawal \
  -H "Authorization: Bearer <accessToken>" \
  -H "Content-Type: application/json" \
  -d '{
    "accountId": "ACC-00000001",
    "amount": 5000,
    "currency": "SGD",
    "bankAccountNumber": "9876543210",
    "bankName": "OCBC Bank",
    "bankCode": "7339",
    "beneficiaryName": "John Doe",
    "withdrawalMethod": "BANK_TRANSFER",
    "purpose": "Personal use"
  }'
```

#### Request body

<ParamField body="accountId" type="string" required>Account ID to withdraw from.</ParamField>
<ParamField body="amount" type="number" required>Withdrawal amount.</ParamField>
<ParamField body="currency" type="string" required>ISO 4217 currency code.</ParamField>
<ParamField body="bankAccountNumber" type="string" required>Destination bank account number.</ParamField>
<ParamField body="bankName" type="string" required>Bank name.</ParamField>
<ParamField body="bankCode" type="string" required>Bank clearing code.</ParamField>
<ParamField body="beneficiaryName" type="string" required>Name on the bank account.</ParamField>
<ParamField body="withdrawalMethod" type="string" required>Withdrawal method (e.g., `BANK_TRANSFER`).</ParamField>
<ParamField body="purpose" type="string">Optional reason for the withdrawal.</ParamField>

#### Response

## Portfolio change requests

Portfolio change requests capture a client's intent to change their investment strategy. They can be self-initiated (client changes strategy directly) or TR-initiated (TR proposes a change, client must confirm).

### POST /v1/portfolio-requests

Create a portfolio change request. Requires the `portfolio-requests:create` permission.

**Authentication**: JWT required (trader/staff role)

```bash
curl -X POST http://localhost:3001/v1/portfolio-requests \
  -H "Authorization: Bearer <accessToken>" \
  -H "Content-Type: application/json" \
  -d '{
    "targetUserId": "USR-00000001",
    "accountId": "ACC-00000001",
    "portfolioId": "PTF-00000001",
    "suggestedPortfolioId": "PTF-00000002",
    "reason": "Client requested lower risk exposure",
    "answers": [
      {"questionId": "QST-00000001", "answer": "capital_preservation"}
    ]
  }'
```

#### Request body

<ParamField body="targetUserId" type="string" required>The client user ID this request is for.</ParamField>
<ParamField body="accountId" type="string" required>Account ID to associate the request with.</ParamField>
<ParamField body="portfolioId" type="string" required>Current portfolio ID.</ParamField>
<ParamField body="suggestedPortfolioId" type="string">Suggested new portfolio template ID.</ParamField>
<ParamField body="reason" type="string">Reason for the change request.</ParamField>
<ParamField body="answers" type="object[]">Updated questionnaire answers submitted alongside the request.</ParamField>

#### Response

The created portfolio request object:

<ResponseField name="id" type="string" required>Request ID.</ResponseField>
<ResponseField name="status" type="string" required>Initial status.</ResponseField>
<ResponseField name="requestType" type="string" required>Request type identifier.</ResponseField>
<ResponseField name="initiatedBy" type="string" required>Role that initiated the request (`CLIENT` or `ADMIN`).</ResponseField>
<ResponseField name="requesterId" type="string" required>User ID of the requester.</ResponseField>
<ResponseField name="targetUserId" type="string" required>User ID of the target client.</ResponseField>
<ResponseField name="createdAt" type="string" required>ISO 8601 timestamp.</ResponseField>

## Trading Representative (TR) endpoints

TR endpoints allow a Trading Representative to act on behalf of a client. Before calling these endpoints, the TR must have selected a client in the `ClientGate`. All TR endpoints require the authenticated user to have the `TradingRepresentative` role — other roles receive `403 Forbidden`.

<Warning>
TR-initiated requests (risk profile and strategy change) create portfolio requests with status `PENDING_CONFIRMATION`. The client must explicitly confirm via `POST /v1/portfolio-requests/:id/confirm` before the change is applied.
</Warning>

### GET /v1/tr/accounts

List all client accounts managed by the authenticated Trading Representative.

**Authentication**: JWT required (TradingRepresentative role only)

```bash
curl http://localhost:3001/v1/tr/accounts \
  -H "Authorization: Bearer <accessToken>"
```

#### Response

<ResponseField name="data" type="object[]" required>

    <ResponseField name="status" type="string">Account status.</ResponseField>

### POST /v1/tr/risk-profile-requests

Submit a completed risk profiling questionnaire on behalf of a client. The backend scores the answers, derives a risk level and recommended portfolio, then creates a portfolio request pending the client's confirmation.

**Authentication**: JWT required (TradingRepresentative role only)

```bash
curl -X POST http://localhost:3001/v1/tr/risk-profile-requests \
  -H "Authorization: Bearer <accessToken>" \
  -H "Content-Type: application/json" \
  -d '{
    "targetUserId": "USR-00000001",
    "answers": [
      {"questionId": "QST-00000001", "answer": "capital_growth"},
      {"questionId": "QST-00000002", "answer": "5_to_10_years"},
      {"questionId": "QST-00000003", "answer": "moderate_risk"}
    ]
  }'
```

#### Request body

<ParamField body="targetUserId" type="string" required>
  The client user ID on whose behalf the questionnaire is being submitted. Must be a client the TR manages.
</ParamField>

<ParamField body="answers" type="object[]" required>
  Array of question-answer pairs.
  <Expandable title="answer properties">
    <ParamField body="answers[].questionId" type="string" required>Question ID.</ParamField>
    <ParamField body="answers[].answer" type="string" required>Selected answer value.</ParamField>
  </Expandable>
</ParamField>

#### Response

<ResponseField name="requestId" type="string" required>Created portfolio request ID (awaiting client confirmation).</ResponseField>
<ResponseField name="submissionId" type="string" required>Questionnaire submission ID.</ResponseField>
<ResponseField name="targetUserId" type="string" required>Client user ID.</ResponseField>
<ResponseField name="suggestedRiskLevel" type="string" required>Computed risk level: `HIGH`, `MEDIUM`, or `LOW`.</ResponseField>
<ResponseField name="riskProfileType" type="string" required>Risk profile classification label.</ResponseField>
<ResponseField name="status" type="string" required>Request status. Will be `PENDING_CONFIRMATION`.</ResponseField>
