---
title: "Portfolio, Accounts, and Instruments Endpoints"
description: "Client API portfolio endpoints: accounts, portfolios, portfolio templates, risk questionnaires, instruments, and holdings on port 3001."
---

The portfolio group of endpoints covers everything a client needs to view and manage their investment position: account details, portfolio holdings, strategy changes, risk profiling questionnaires, and available instruments. All endpoints require a valid JWT access token in the `Authorization: Bearer` header.

<Note>
Trading Representative (TR) users can pass an optional `userId` query parameter to access a client's data on their behalf. The backend resolves this via `TrAccessService` and validates that the TR has access to the specified client account.
</Note>

---

## Accounts

### GET /v1/accounts

List all accounts belonging to the authenticated client.

**Authentication**: JWT required

```bash
curl http://localhost:3001/v1/accounts \
  -H "Authorization: Bearer <accessToken>"
```

### Response

<ResponseField name="data" type="object[]" required>
  Array of account objects.
  <Expandable title="account properties">
    <ResponseField name="id" type="string">Account ID (e.g., `ACC-00000001`).</ResponseField>
    <ResponseField name="name" type="string">Display name of the account.</ResponseField>
    <ResponseField name="userId" type="string">Owning user ID.</ResponseField>
    <ResponseField name="status" type="string">Account status (`active`, `inactive`, `closed`).</ResponseField>
    <ResponseField name="bankAccountNumber" type="string">Linked bank account number, may be null.</ResponseField>
    <ResponseField name="bankName" type="string">Linked bank name, may be null.</ResponseField>
    <ResponseField name="createdAt" type="string">ISO 8601 timestamp.</ResponseField>
    <ResponseField name="updatedAt" type="string">ISO 8601 timestamp.</ResponseField>
  </Expandable>
</ResponseField>

### PATCH /v1/accounts/:id

Update the account name or linked bank details for the authenticated user.

**Authentication**: JWT required

```bash
curl -X PATCH http://localhost:3001/v1/accounts/ACC-00000001 \
  -H "Authorization: Bearer <accessToken>" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "My Retirement Account",
    "bankAccountNumber": "0123456789",
    "bankName": "DBS Bank",
    "bankCode": "7171",
    "beneficiaryName": "John Doe"
  }'
```

#### Path parameters

<ParamField path="id" type="string" required>
  Account ID.
</ParamField>

#### Request body

<ParamField body="name" type="string">
  New display name for the account.
</ParamField>

<ParamField body="bankAccountNumber" type="string">
  Bank account number for withdrawals.
</ParamField>

<ParamField body="bankName" type="string">
  Name of the bank.
</ParamField>

<ParamField body="bankCode" type="string">
  Bank clearing code.
</ParamField>

<ParamField body="beneficiaryName" type="string">
  Name of the account beneficiary.
</ParamField>

#### Response

Returns the updated account object.

## Portfolios

### GET /v1/portfolios/my-portfolios

Retrieve all portfolios belonging to the authenticated user, with their latest portfolio request status. Supports pagination and sorting.

**Authentication**: JWT required

```bash
curl "http://localhost:3001/v1/portfolios/my-portfolios?page=1&limit=20" \
  -H "Authorization: Bearer <accessToken>"
```

#### Query parameters

<ParamField query="page" type="number" default="1">
  Page number for pagination.
</ParamField>

<ParamField query="limit" type="number" default="20">
  Number of results per page.
</ParamField>

<ParamField query="sortBy" type="string">
  Field to sort by (e.g., `createdAt`).
</ParamField>

<ParamField query="sortOrder" type="string">
  Sort direction: `ASC` or `DESC`.
</ParamField>

<ParamField query="userId" type="string">
  TR only. The client user ID to fetch portfolios on behalf of.
</ParamField>

#### Response

<ResponseField name="data" type="object[]" required>
  Paginated array of portfolio objects.
  <Expandable title="portfolio properties">
    <ResponseField name="id" type="string">Portfolio ID (e.g., `PTF-00000001`).</ResponseField>
    <ResponseField name="name" type="string">Portfolio name, may be null.</ResponseField>
    <ResponseField name="description" type="string">Portfolio description, may be null.</ResponseField>
    <ResponseField name="riskLevel" type="string">Risk level: `HIGH`, `MEDIUM`, or `LOW`.</ResponseField>
    <ResponseField name="userId" type="string">Owner user ID.</ResponseField>
    <ResponseField name="status" type="string">Portfolio status: `draft`, `active`, `rebalancing`, or `closed`.</ResponseField>
    <ResponseField name="isPrimary" type="boolean">Whether this is the user's primary portfolio.</ResponseField>
    <ResponseField name="requestId" type="string">Latest portfolio request ID, may be null.</ResponseField>
    <ResponseField name="requestStatus" type="string">Latest request status (e.g., `PENDING_APPROVAL`), may be null.</ResponseField>
    <ResponseField name="submission" type="object">Associated questionnaire submission, if any.</ResponseField>
    <ResponseField name="details" type="object[]">Asset allocation breakdown for the portfolio.</ResponseField>
    <ResponseField name="createdAt" type="string">ISO 8601 timestamp.</ResponseField>
    <ResponseField name="updatedAt" type="string">ISO 8601 timestamp.</ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="total" type="number" required>Total count of matching portfolios.</ResponseField>
<ResponseField name="page" type="number" required>Current page.</ResponseField>
<ResponseField name="limit" type="number" required>Page size.</ResponseField>
<ResponseField name="totalPages" type="number" required>Total number of pages.</ResponseField>

## Questionnaires

The risk profiling questionnaire is used during onboarding and strategy changes. Clients (or TRs on their behalf) answer a set of questions. The system scores their answers and assigns a risk profile (`HIGH`, `MEDIUM`, or `LOW`) which maps to a recommended portfolio strategy.

<Warning>
Each user is limited to a maximum of 3 questionnaire submissions. Subsequent submissions overwrite the risk profile recommendation.
</Warning>

### GET /v1/questionnaires/:questionGoal/questions

Retrieve all questions for a given questionnaire goal.

**Authentication**: JWT required

```bash
curl "http://localhost:3001/v1/questionnaires/RISK_PROFILE/questions" \
  -H "Authorization: Bearer <accessToken>"
```

#### Path parameters

<ParamField path="questionGoal" type="string" required>
  The questionnaire goal type. Use `RISK_PROFILE` for the standard risk profiling questionnaire.
</ParamField>

#### Response

Array of question objects:

<ResponseField name="id" type="string">Question ID (e.g., `QST-00000001`).</ResponseField>
<ResponseField name="questionGoal" type="string">Goal this question belongs to.</ResponseField>
<ResponseField name="questionName" type="string">Short label for the question.</ResponseField>
<ResponseField name="questionDetail" type="string">Full question text shown to the user.</ResponseField>
<ResponseField name="options" type="object[]">Array of answer options with their values and score weights.</ResponseField>
<ResponseField name="order" type="number">Display order within the questionnaire.</ResponseField>

---

### POST /v1/questionnaires/:questionGoal/submissions

Submit answers for a questionnaire. The backend scores the answers and derives a risk profile and recommended portfolio.

**Authentication**: JWT required

```bash
curl -X POST "http://localhost:3001/v1/questionnaires/RISK_PROFILE/submissions" \
  -H "Authorization: Bearer <accessToken>" \
  -H "Content-Type: application/json" \
  -d '{
    "answers": [
      {"questionId": "QST-00000001", "answer": "capital_growth"},
      {"questionId": "QST-00000002", "answer": "5_to_10_years"},
      {"questionId": "QST-00000003", "answer": "high_risk_tolerance"}
    ],
    "goalSetup": {
      "targetAmount": 100000,
      "investmentHorizon": 10
    }
  }'
```

#### Path parameters

<ParamField path="questionGoal" type="string" required>
  The questionnaire goal. Use `RISK_PROFILE` for risk profiling.
</ParamField>

#### Request body

<ParamField body="answers" type="object[]" required>
  Array of question-answer pairs.
  <Expandable title="answer properties">
    <ParamField body="answers[].questionId" type="string" required>
      The question ID from `GET /v1/questionnaires/:questionGoal/questions`.
    </ParamField>
    <ParamField body="answers[].answer" type="string" required>
      The selected answer value.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="goalSetup" type="object">
  Optional investment goal parameters captured during the goal-setup wizard step.
</ParamField>

#### Response

<ResponseField name="id" type="string">Submission ID (e.g., `SUB-00000001`).</ResponseField>
<ResponseField name="questionGoal" type="string">The questionnaire goal.</ResponseField>
<ResponseField name="userId" type="string">Submitting user ID.</ResponseField>
<ResponseField name="answerDate" type="string">ISO 8601 submission timestamp.</ResponseField>
<ResponseField name="riskProfile" type="object">
  Derived risk profile.
  <Expandable title="riskProfile properties">
    <ResponseField name="riskLevel" type="string">Computed risk level: `HIGH`, `MEDIUM`, or `LOW`.</ResponseField>
    <ResponseField name="riskProfileType" type="string">Risk profile classification label.</ResponseField>
  </Expandable>
</ResponseField>
<ResponseField name="portfolio" type="object">Recommended portfolio template matching the risk profile, may be null.</ResponseField>
