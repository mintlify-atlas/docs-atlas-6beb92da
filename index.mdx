UTRADE Robo Advisory is a wealth management platform built for UOB Kay Hian (Cambodia). It delivers automated portfolio recommendations, questionnaire-based risk profiling, trading execution via FNZ/Tapico, and a dual-portal experience — a client-facing web portal for self-service investment management and an admin dashboard for back-office operations, approvals, and user management.

## Platform overview

UTRADE Robo Advisory consists of five repositories working together:

| Service | Role | Port |
|---|---|---|
| `ms-robo-admin` | Admin REST API — schema owner | 3002 |
| `ms-robo-web` | Client REST API — schema consumer | 3001 |
| `mfe-robo-admin` | Admin dashboard (React + Vite) | 5173 |
| `mfe-robo-web` | Client web portal (React + Vite) | 5174 |
| `ms-robo-dataloader` | Data loading service (placeholder) | — |

## Key capabilities

<CardGroup cols={2}>
  <Card title="Risk profiling" icon="chart-pie">
    Questionnaire-based assessment maps client answers to risk profiles and recommends matching portfolio strategies automatically.
  </Card>
  <Card title="Portfolio management" icon="briefcase">
    Model portfolio templates define target allocations. Client portfolios are created, tracked, and updated through approval workflows.
  </Card>
  <Card title="Trading execution" icon="arrow-right-arrow-left">
    Orders are placed and settled via the FNZ/Tapico integration. Holdings, valuations, and snapshots are tracked in real time.
  </Card>
  <Card title="Approval workflows" icon="circle-check">
    Funding deposits, withdrawals, and portfolio strategy changes flow through a structured approval process managed by back-office staff.
  </Card>
  <Card title="SSO authentication" icon="key">
    Keycloak OIDC with PKCE provides single sign-on for both portals, with fallback username/password login.
  </Card>
  <Card title="Data synchronization" icon="rotate">
    Client user data is synced automatically from the SUS8 Redshift datalake every day at 2 AM.
  </Card>
</CardGroup>

## Get started

<Steps>
  <Step title="Set up local infrastructure">
    Start PostgreSQL, Keycloak, and SonarQube using Docker Compose. See [Local Setup](/local-setup) for the full walkthrough.
  </Step>
  <Step title="Start the backend services">
    Run `bun run start:dev` inside `ms-robo-admin` (port 3002) and `ms-robo-web` (port 3001). Both expose Swagger docs at `/docs`.
  </Step>
  <Step title="Start the frontend apps">
    Run `bun run dev` inside `mfe-robo-admin` (port 5173) and `mfe-robo-web` (port 5174). Log in with the test credentials from [Local Setup](/local-setup).
  </Step>
  <Step title="Explore the API">
    Visit the [Admin API Reference](/api/admin/auth) or [Client API Reference](/api/client/auth) to start making calls.
  </Step>
</Steps>
